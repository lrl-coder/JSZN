## 如果仍然超时

如果这两个测试仍然超时，可以考虑以下方案：

### 方案1：跳过这些测试（推荐用于快速测试）

在测试方法上添加 `@Ignore` 注解：

```java
@Ignore("长时间运行测试，在快速测试中跳过")
@Test(timeout = 300000)
public void testVeryLargeNumberOfOrders() {
    // ...
}
```

### 方案2：进一步减少规模

如果必须运行这些测试，可以进一步减少：
- 订单数量：20 → 10
- 工件数量：10 → 5
- 种群大小：15 → 10
- 迭代次数：8 → 5

### 方案3：使用Maven配置跳过

在运行测试时跳过这些测试：

```bash
# 运行除BoundaryCaseTest外的所有测试
mvn test -Dtest=!BoundaryCaseTest

# 或者只运行特定测试类
mvn test -Dtest=OrderTest,ProductTest,ScheduleDataTest,TimeCostUtilTest,GASchedulerTest
```
